## 🧠 2. 为什么说 “LLM 能力是有限的”？

LLM（大模型）本质上是：

> 👉 一个“概率预测器”（预测下一个 token）

它有几个天然限制：

### ❌ 不能直接做的事

* 不能访问实时数据（除非接 API）
* 不能执行代码
* 不能操作系统 / 浏览器
* 长期记忆很弱（上下文有限）

---

## 🧩 3. 所以才需要三件套

你提到的三件东西，其实可以这样拆：

### ① Prompt（大脑的“指令设计”）

* 决定模型怎么思考
* 比如：

  * Chain-of-Thought
  * ReAct（思考 + 行动）
  * System Prompt

👉 作用：**让模型“更聪明一点”**

---

### ② Tools（让模型“有手有脚”）

也就是你说的“工具能力”，比如：

* 搜索（Web Search）
* 数据库查询
* 调接口（天气、订单）
* 执行代码（Python / JS）
* 操作浏览器（RPA）

👉 作用：**让模型从“会说”变成“能做”**

---

### ③ Engineering（你说的工程能力 👉 关键！）

这其实是很多人忽略但最重要的部分：

#### 前端工程

* Chat UI（类似 ChatGPT）
* 流式输出（SSE / WebSocket）
* 状态管理（多轮对话）

#### 后端工程

* Prompt 编排（Prompt Template）
* Tool 调度（Function Calling）
* Agent 状态机
* 记忆系统（Memory）
* RAG（检索增强生成）

#### 系统设计

* 限流 / 重试
* 成本控制（Token优化）
* 日志 & 监控
* 多 Agent 协作

👉 作用：**把 AI 从 Demo 变成产品**

---

## 🤖 4. 一个完整 Agent 的真实结构（帮你升维）

你可以这样理解一个现代 Agent：

```
用户输入
   ↓
Prompt（任务描述 + 上下文）
   ↓
LLM 思考（要不要用工具？）
   ↓
Tool 调用（API / DB / Code）
   ↓
结果返回给 LLM
   ↓
LLM 总结输出
   ↓
前端展示
```
