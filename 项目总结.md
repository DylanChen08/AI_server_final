# AI_server_final / A2UI Server — 项目总结

本文档概括本仓库的定位、主要模块与运行方式，便于快速了解与交接。细节以各子目录说明与源码为准。

---

## 1. 项目定位

本仓库是一个以 **Node.js + TypeScript** 为主的 **AI 服务端与实验代码集合**，包名为根目录 `a2ui`，工作区核心在 **`server`**（`@a2ui/server`）。主要包含三类能力：

1. **MCP（Model Context Protocol）服务**：通过 stdio 向 Cursor 等客户端暴露工具（如根据 API 路径读取 IDL）。
2. **多智能体（Multi-Agent）演示**：主 Agent 做规划/编排，子 Agent 执行具体任务；交互通过终端 TUI 与 OpenAI 兼容 API。
3. **OpenAI 兼容调用层**：统一封装聊天与函数（工具）调用循环，支持多家厂商（通义、Moonshot、OpenAI 等）通过环境变量切换。

当前 `server` 下 **未发现已提交的 `dist/` 构建产物**；`package.json` 中的 `start` 指向编译后的 `dist/index.js`，与现有以 **ts-node / 直接入口脚本** 运行 MCP 与 Agent 的方式并存，使用前请按下文「运行方式」选择合适入口。

## 2. 仓库结构（要点）

| 路径 | 说明 |
|------|------|
| `server/` | 主服务端包：MCP、多智能体、OpenAI 封装、配置与依赖 |
| `server/src/mcp/` | MCP 实现：`main.ts` 入口，`server.ts` 注册工具；`idl/` 为接口描述 JSON；`resources/` 等为业务/mock 相关资源 |
| `server/src/openAI/` | API Key、模型与 Base URL 解析（`key.ts`）、`openai.ts` 请求封装、`tui.ts` 终端对话与工具循环 |
| `server/src/multi-agent/` | **主从多智能体**：`master-agent`（待办 + 调用子 Agent）；`agents/write-file-agent`、`agents/code-review-agent` |
| `server/src/multi-agent-master-agent/` | **简化版主 Agent**：主 Agent 仅挂载读写文件类工具（无子进程子 Agent 编排） |
| `server/run-mcp.cjs` | 固定工作目录并用 ts-node 拉起 MCP，供 Cursor 等项目级 MCP 配置调用 |
| `.cursor/mcp.json` | Cursor 项目级 MCP 配置（与 `MCP_Cursor_接入说明.md` 配套） |
| `workspace/test/` | 与主 `server` 平行的测试/副本目录（结构类似，按需使用） |
| 根目录其他 `*.md` | 架构说明、ReAct、Agent Loop 等笔记类文档 |

根目录 `package.json` 的 `build` 脚本中仍出现 `packages/common`、`web` 等 workspace 名称，与当前仅声明 `server` workspace 的布局可能不一致；以实际目录与 `server/package.json` 为准。

## 3. 核心模块说明

### 3.1 MCP（`server/src/mcp`）

- **传输**：Stdio（标准输入输出），非 HTTP。
- **工具示例**：`generate_mock` — 根据 `apiUrl` 映射到 `server/src/mcp/idl/*.json` 并返回 IDL 文本与提示（当前设计多为「把 IDL 交给上层再生成 mock」，而非直接返回最终业务 JSON）。
- **接入 Cursor**：详见根目录 [`MCP_Cursor_接入说明.md`](./MCP_Cursor_接入说明.md)（`run-mcp.cjs`、schema、同名配置冲突、自检命令等）。

### 3.2 多智能体（`server/src/multi-agent`）

- **主 Agent**：加载中文系统提示词（`master-agent/prompt`）、工具（`master-agent/tools`），通过 `openAI/tui.ts` 启动交互式会话。
- **工具能力概要**：待办管理（增删改查）+ `callSubAgent`，按类型派生到子 Agent 目录（如写文件、代码审查），子 Agent 各自维护 `prompt` 与 `tools`。
- **入口**：`server/src/multi-agent/main.ts`（需配置 `server/.env` 中的 API Key 等）。

### 3.3 简化主 Agent（`server/src/multi-agent-master-agent`）

- 主 Agent 工具为 **`writeFile` / `readFile`**（相对 `master-agent` 工作目录），同样走 TUI。
- **入口**：`server/src/multi-agent-master-agent/main.ts`。

### 3.4 OpenAI 兼容层（`server/src/openAI`）

- **密钥与厂商**：在 `server/.env` 中配置 `DASHSCOPE_API_KEY`、`MOONSHOT_API_KEY`（或 `KIMI_API_KEY`）、`OPENAI_API_KEY` 等之一；可选 `OPENAI_BASE_URL`、`OPENAI_MODEL` / `LLM_MODEL`。
- **Agent 步数上限**：TUI 内工具调用循环受 `AGENT_MAX_STEPS` 约束（默认 16，可改环境变量）。

## 4. 环境与运行方式

1. **安装依赖**：在仓库根目录执行 `npm install`（或按团队使用的包管理器安装 workspace）。
2. **环境变量**：在 **`server/.env`** 配置 LLM 相关变量（见上节）；Agent 入口会加载该路径附近的 `.env`。
3. **MCP（终端自检）**：
   ```bash
   node /path/to/AI_server_final/server/run-mcp.cjs
   ```
   预期 stderr 出现类似 `####mcp server run`，进程保持运行等待 stdio 协议。
4. **多智能体 TUI**：使用 `ts-node` 等方式执行对应 `main.ts`（例如 `server/src/multi-agent/main.ts`），需已配置 API Key。
5. **编译**：`server` 包内 `npm run build` 执行 `tsc`；若需 `npm start`，需先成功生成 `dist` 且存在与 `main` 字段一致的入口。

## 5. 技术栈摘要

- **语言**：TypeScript
- **LLM 客户端**：`openai` 官方 SDK（OpenAI 兼容端点）
- **MCP**：`@modelcontextprotocol/sdk`
- **终端交互**：`inquirer`、`chalk`
- **其他依赖**：`dotenv`、`zod`；`server/package.json` 中声明了 `koa` 等，是否与当前 `src` 使用一致以代码为准
