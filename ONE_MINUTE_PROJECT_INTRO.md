# 1 分钟项目介绍（面试口述稿）

我做了一个基于 LLM 的 **Multi-Agent 编排系统**，核心目标是把“写代码、读代码、审查代码”这类工程任务拆成可控的执行单元，提高自动化效率和可维护性。架构上采用 **Master Agent + Sub-Agent**：Master 负责在 TUI 里与用户交互、拆解任务并用 todo 管理执行顺序；当需要落地动作时，Master 通过 `spawn` 启动独立进程的子 Agent 来执行具体能力，比如 **WriteFile Agent** 只负责写文件，**CodeReview Agent** 只负责读文件并输出审查建议。整个系统通过 OpenAI 的 function calling 实现 **Agent loop**：模型生成→触发工具调用→本地执行工具→把结果回写到对话历史→再让模型继续，直到没有新的工具调用或达到步数上限。这样做的好处是 **上下文隔离更清晰（进程/会话/能力边界）**、扩展新 Agent 成本低，同时也能通过 step 上限、日志与错误分流提升稳定性。后续我计划进一步加强安全边界，比如限制文件路径越界、把“完成信号”从字符串升级为结构化协议，并补齐超时与重试策略。

## 追问可补充（可选一句）

- 如果面试官问“你做的最关键点是什么”：我认为最关键是把执行能力做成工具与子进程隔离，让编排和执行解耦，既能控风险也能扩展能力。
