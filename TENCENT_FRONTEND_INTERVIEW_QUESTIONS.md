# 腾讯中高级前端面试问题清单（结合本项目）

> 目标：帮助你围绕当前项目（Multi-Agent + 工具调用 + 工程化）准备中高级前端面试。  
> 用法：每个问题先答“结论 + 方案 + trade-off + 落地细节”，再补 1 个项目例子。

---

## 一、项目与架构理解（高频）

1. 这个项目的核心业务目标是什么？你负责了哪一块？
2. 为什么采用 Master-Agent + Sub-Agent 的架构，而不是单 Agent？
3. 你如何定义“编排层”和“执行层”的职责边界？
4. 如果让你重构当前架构，第一优先级改什么？为什么？
5. 这个架构相比传统前后端分层，最大收益和最大成本分别是什么？
6. 你如何向非技术同学解释 Multi-Agent 的价值？

**常见追问**
- 你做过哪些架构取舍？
- 有没有被你否掉的方案？为什么？

---

## 二、上下文管理与隔离（项目强相关）

1. 你们是如何做上下文隔离的？（进程、会话、工具权限、协议）
2. 为什么说当前是“逻辑隔离”而不是“强安全隔离”？
3. 如果子 Agent 拿到越权路径，你怎么防止目录逃逸？
4. 子 Agent 的结果回传为什么不能只靠字符串（如 `finish`）判断？
5. 多个子 Agent 并发时，如何避免上下文串扰和任务污染？
6. 你如何设计可扩展的任务协议（task id、状态、结果）？

**常见追问**
- 给一个你会立刻改的安全点。
- 如果线上出现串任务，你怎么定位？

---

## 三、前端工程化与可维护性（腾讯很看重）

1. 你如何设计项目目录结构让多人协作成本最低？
2. TypeScript 在这个项目里解决了什么问题？还有哪些类型盲区？
3. 你如何约束工具调用参数，避免运行时参数错位？
4. 你如何做日志分级（debug/info/warn/error）和可观测性埋点？
5. 你如何定义“可维护代码”？你在项目中有哪些具体实践？
6. 如何避免“提示词写死在代码里”带来的维护风险？

**常见追问**
- 你遇到过最难维护的一段代码是什么，怎么治理的？
- 如何推动团队统一代码规范？

---

## 四、性能与稳定性（中高级分水岭）

1. Agent 循环为什么要设置 `max steps`？阈值怎么定？
2. 流式输出在体验和性能上的收益是什么？
3. 如果 LLM 响应慢或波动大，你有哪些降级策略？
4. 子进程卡死如何检测？超时、重试、熔断怎么设计？
5. 如何避免工具调用导致死循环或雪崩？
6. 你会如何做“慢调用分析”和“失败归因”？

**常见追问**
- 线上超时率突然升高，你的排查顺序是什么？
- 如何在不改业务逻辑情况下提升稳定性？

---

## 五、安全与风控（高优先级）

1. 文件读写工具有哪些安全风险？你做了哪些防护？
2. 如何防 Prompt Injection 影响工具调用？
3. 对“工具白名单”你会怎么设计和落地？
4. 你如何处理敏感信息（API Key、日志脱敏、错误回传）？
5. 如何避免 LLM 输出影响系统执行安全（如命令注入）？
6. 如果要做审计，你会记录哪些关键字段？

**常见追问**
- 给一个真实的攻击面和修复方案。
- 如何证明你的方案有效？

## 六、LLM 工具调用机制（项目深挖）

4. 工具结果写回会话时，如何保证 `tool_call_id` 一致性？
5. 如何防止模型“幻觉调用”不存在的工具？
6. 模型切换（OpenAI/Moonshot/Dashscope）时怎么保持行为稳定？

**常见追问**
- 你遇到过最棘手的一次工具调用 bug 是什么？

## 七、任务编排与状态机（架构能力题）

1. 为什么要有 TodoManager？它解决了什么问题？

3. 如何升级为支持依赖图的并发调度？
4. 如何定义任务状态机（pending/in_progress/completed/failed/retry）？
5. 怎么保证任务幂等，防止重复执行？

**常见追问**

## 九、前端视角延展题（腾讯常问“端到端思维”）

## 十一、可直接准备的“项目亮点回答模板”

1. **架构亮点**：主控编排与执行分离，职责清晰，具备扩展子 Agent 的能力。  
2. **稳定性亮点**：有 Agent step 上限、子进程回收、基础错误分流。  
3. **工程亮点**：模块化拆分、工具描述清晰、便于新增能力。  
4. **安全改进点**：计划补齐路径沙箱、结构化完成信号、超时与重试机制。  
5. **前端延展点**：可视化执行链路、流式交互体验、可观测性面板。

---

## 十二、面试前 30 分钟速刷清单

1. 用 1 分钟讲清项目目标、你的职责、核心结果。  
2. 用 2 分钟讲清 Master/Sub-Agent 架构与上下文隔离。  
3. 准备 2 个“线上风险 -> 修复方案 -> 效果”的案例。  
4. 准备 1 个“性能优化”案例和 1 个“安全治理”案例。  
5. 准备 1 个“带团队协作推进”的故事（有冲突、有结果、有复盘）。

---

## 十三、加分答题结构（建议固定套路）

答每道中高级问题时，尽量按这 5 步：

1. **先给结论**（一句话）  
2. **再讲方案**（核心机制）  
3. **说取舍**（为什么不用别的）  
4. **落到项目细节**（你做了什么）  
5. **补优化方向**（下一步怎么做）

这样会显得你不仅会做，还具备架构判断与工程 owner 视角。
